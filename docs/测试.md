# 实验室设备报修管理系统测试说明

## 1. 测试目标

本测试文档用于验证当前阶段新增能力：

- 基于数据库账号密码登录。
- BCrypt 密码校验。
- 普通用户公开注册，注册后进入待审核状态。
- 管理员创建四类用户：系统管理员、实验室管理员、维修人员、报修人员。
- 管理员审核、拒绝、禁用、启用用户。
- 不同账号状态下的登录错误提示。
- 个人资料查看、修改。
- 头像上传并返回数据库保存路径。
- 前端注册页、用户管理页、个人资料页基础构建通过。

## 2. 测试环境准备

### 2.1 数据库准备

数据库名：

```sql
lab-equipment-repair
```

已建库时执行增量脚本：

```sql
source docs/alter_user_registration_profile.sql;
```

脚本会完成：

- `user.status` 中文注释更新。
- `user.avatar_path` 字段补充。
- `idx_user_created_at` 索引补充。

### 2.2 后端准备

后端默认配置：

```properties
server.port=18080
spring.datasource.url=jdbc:mysql://127.0.0.1:3306/lab-equipment-repair
spring.datasource.username=root
spring.datasource.password=123456
```

启动后端后，`DataInitializer` 会自动创建四个基础账号：

| 账号 | 密码 | 角色 |
| --- | --- | --- |
| admin | 123456 | ADMIN |
| manager | 123456 | LAB_MANAGER |
| repairer | 123456 | REPAIRER |
| reporter | 123456 | REPORTER |

### 2.3 前端准备

安装依赖后执行：

```bash
cd frontend
npm.cmd run build
```

启动开发服务：

```bash
cd frontend
npm.cmd run dev -- --host 127.0.0.1 --port 5173
```

访问：

```text
http://127.0.0.1:5173/
```

## 3. 自动化接口测试脚本

脚本位置：

```text
tests/api_smoke_test.py
```

执行命令：

```bash
python tests/api_smoke_test.py
```

指定接口地址和管理员账号：

```bash
python tests/api_smoke_test.py --base-url http://127.0.0.1:18080/api --admin admin --password 123456
```

脚本使用 Python 标准库实现，不依赖 `requests` 或 `pytest`。

## 4. 自动化测试覆盖范围

### 4.1 登录成功

输入：

- `admin / 123456`

期望：

- HTTP `200`
- 业务 `code = 200`
- 返回 `data.token`
- 返回当前用户信息

### 4.2 密码错误

输入：

- `admin / wrong-password`

期望：

- HTTP `401`
- 业务 `code = 401`
- `message` 包含 `账号或密码错误`

### 4.3 普通用户注册

接口：

```text
POST /api/auth/register
```

期望：

- 注册成功。
- `role = REPORTER`
- `status = PENDING`
- 未登录也可以访问该接口。

### 4.4 待审核用户登录

输入：

- 刚注册的 `PENDING` 用户

期望：

- HTTP `403`
- 业务 `code = 403`
- `message` 包含 `账号待管理员审核`

### 4.5 管理员创建用户

接口：

```text
POST /api/users
```

期望：

- 仅管理员可访问。
- 可创建 `ADMIN / LAB_MANAGER / REPAIRER / REPORTER`。
- 自动使用 BCrypt 加密密码。
- 默认 `status = ACTIVE`。

### 4.6 管理员审核用户

接口：

```text
PUT /api/users/{id}/approve
```

期望：

- `PENDING` 用户变为 `ACTIVE`。
- 审核后用户可登录。

### 4.7 管理员禁用和启用用户

接口：

```text
PUT /api/users/{id}/disable
PUT /api/users/{id}/enable
```

期望：

- 禁用后用户登录返回 `账号已被禁用`。
- 启用后用户可重新登录。

### 4.8 个人资料修改

接口：

```text
PUT /api/profile
```

期望：

- 当前登录用户可修改 `realName` 和 `phone`。
- 响应中的用户资料同步更新。

### 4.9 头像上传

接口：

```text
POST /api/profile/avatar
```

期望：

- 支持 `image/png`、`image/jpeg`、`image/webp`。
- 文件保存到后端 `uploads/avatars`。
- 数据库保存路径形如 `/uploads/avatars/xxx.png`。
- 响应 `avatarPath` 以 `/uploads/avatars/` 开头。

## 5. 边界测试清单

### 5.1 用户名边界

| 场景 | 输入 | 期望 |
| --- | --- | --- |
| 用户名为空 | `""` | 参数校验失败 |
| 用户名长度小于 3 | `"ab"` | 参数校验失败 |
| 用户名长度等于 3 | `"abc"` | 可注册或创建 |
| 用户名长度等于 50 | 50 个字符 | 可注册或创建 |
| 用户名重复 | 已存在账号 | 返回 `账号已存在` |

### 5.2 密码边界

| 场景 | 输入 | 期望 |
| --- | --- | --- |
| 密码为空 | `""` | 参数校验失败 |
| 密码长度小于 6 | `"12345"` | 参数校验失败 |
| 密码长度等于 6 | `"123456"` | 可注册或创建 |
| 密码错误 | 任意错误密码 | 返回 `账号或密码错误` |

### 5.3 角色边界

| 场景 | 输入 | 期望 |
| --- | --- | --- |
| 管理员创建 `ADMIN` | `ADMIN` | 创建成功 |
| 管理员创建 `LAB_MANAGER` | `LAB_MANAGER` | 创建成功 |
| 管理员创建 `REPAIRER` | `REPAIRER` | 创建成功 |
| 管理员创建 `REPORTER` | `REPORTER` | 创建成功 |
| 非法角色 | `UNKNOWN` | 返回 `用户角色不合法` |
| 普通注册传角色 | 任意角色 | 后端忽略，固定为 `REPORTER` |

### 5.4 状态边界

| 状态 | 登录期望 |
| --- | --- |
| `PENDING` | 返回 `账号待管理员审核` |
| `ACTIVE` | 登录成功 |
| `REJECTED` | 返回 `注册申请已被拒绝` |
| `DISABLED` | 返回 `账号已被禁用` |
| 其他异常状态 | 返回 `账号状态异常` |

### 5.5 权限边界

| 场景 | 期望 |
| --- | --- |
| 未登录访问 `/api/auth/register` | 允许 |
| 未登录访问 `/api/users` | 拒绝 |
| 非管理员访问 `/api/users` | 拒绝 |
| 管理员访问 `/api/users` | 允许 |
| 登录用户访问 `/api/profile` | 允许 |
| 未登录访问 `/api/profile` | 拒绝 |

### 5.6 头像上传边界

| 场景 | 期望 |
| --- | --- |
| 上传 PNG | 成功 |
| 上传 JPG | 成功 |
| 上传 WEBP | 成功 |
| 上传空文件 | 返回 `请选择头像文件` |
| 上传非图片 MIME | 返回 `头像仅支持 JPG、PNG、WEBP 格式` |
| 路径穿越文件名 | 后端生成 UUID 文件名，不使用原始文件名 |

## 6. 手工前端测试流程

1. 打开 `http://127.0.0.1:5173/login`。
2. 输入错误密码，确认页面显示 `账号或密码错误`，不是 `系统繁忙`。
3. 点击注册入口，提交一个普通用户注册。
4. 使用刚注册账号登录，确认提示 `账号待管理员审核`。
5. 使用 `admin / 123456` 登录。
6. 进入用户管理，确认待审核用户可通过或拒绝。
7. 创建一个维修人员账号，确认状态为启用。
8. 禁用该账号，确认该账号无法登录。
9. 重新启用该账号，确认可以登录。
10. 进入个人资料页，修改姓名和电话。
11. 上传头像，刷新后确认头像仍可展示。

## 7. 当前已知限制

- 自动化脚本会创建临时用户，账号名带时间戳，不会自动删除。
- 后端编译和测试依赖 Maven 下载，若本机 Maven 依赖未缓存且网络不可用，`mvn compile` 会失败。
- 脚本是接口冒烟测试，不替代完整单元测试和浏览器自动化测试。
- 头像上传测试只验证接口返回和路径格式，不直接检查数据库字段值，需要通过数据库或页面展示进一步确认。

## 8. 通过标准

满足以下条件可认为当前阶段测试通过：

- 数据库增量脚本执行成功。
- 后端启动成功。
- `python tests/api_smoke_test.py` 全部 `[PASS]`。
- `npm.cmd run build` 成功。
- 手工前端流程无明显页面错误或接口错误。
